# 个人 AI 投资分析团队 · v1.0 全景架构蓝图

> 版本：v1.0-blueprint（目标态）｜更新：2026-06-19
> 性质：**目标架构**，描述系统"长成什么样"，不代表现在全建。每模块标注阶段（P0/P1/P2/P3）。
> 一句话使命：持续产出**高质量投资认知与研究产出**，并让"我"这个投资者本身随时间变强——不替我决策、不给买卖结论。
> 现状：**全新搭建**（旧的 Z哥 skill 已废弃，不作为基础）。

---

## 0. 设计原则

1. **最小可用优先**：能用模板+提示词解决的不写代码；大部分"Agent"先是角色，不是独立程序。
2. **认知多样性 + 免疫系统**：必须有强制**反方**与独立**风险**两道闸，避免单一世界观高度自洽地一起错。
3. **三防红线**：防幻觉（重要数据/结论带来源）、防过拟合（量化策略过验证）、防冲动（想法必须过反方+风险才进决策日志）。
4. **以"我"为中心**：系统不只分析市场，还分析**我自己**——记录我的判断、评估对错、识别我的偏差，形成个人认知档案。
5. **可移植 / 可复盘**：Git 为真相源，多机可拉；所有判断可追溯。
6. **人机边界**：真实买卖、仓位/杠杆建议、未验证结论 → 永不自动化。


**关键设计判断**：四个团队不是孤立的，它们靠**两条暗线**串起来——
- **能力2(判断) + 能力4(持仓) = 绩效归因**：把"我市场判断对不对"和"我操作执行对不对"分开评估。
- **能力3(信息) 喂给 T1/T2**：信息不是堆在一起，而是经信噪比过滤后流向研究和判断。

## 2. 四个业务团队

### T1 · 投资分析团队（个股深度分析）｜P1
- **目标**：对单只股票做多维深度分析，产出可执行的研究结论。
- **成员（角色）**：个股研究、产业链/板块研究、**反方观点**、**风险控制**。
- **输入**：股票代码/名称 + 当前市场环境基调。
- **输出**：研究备忘录（多空逻辑、关键数据带来源、证伪条件、不确定性）。
- **流程**：研究 → 反方强制介入 → 风险审查 → 总编汇总。**不给买卖结论**，只给"研究结论 + 入场/离场/证伪条件"，是否执行由我拍板。
- **数据需求**：高（行情、财务、资金、新闻）。

### T2 · 自我提升团队（判断校准）⭐ P0
> 这是这套系统区别于普通投研工具的灵魂，且**几乎不需要外部数据就能启动**。
- **目标**：记录我每天的判断 → 到期回看对错 → 和我讨论 → 长期识别我的认知偏差。
- **成员**：判断记录、校准评分、偏差检测、复盘讨论主持。
- **判断记录字段**：时间 / 判断内容 / 依据 / **置信度(0-100%)** / 时间跨度 / **证伪条件** / 标的或主题。
- **到期评分**：拉取实际结果 → 标记 对/错/部分 → 用**校准分数（类 Brier score）**衡量"你说 80% 把握时，是否真有 80% 兑现"。
- **偏差检测**：跨时间统计——你是否过度自信？追涨杀跌？对某类标的系统性看错？
- **产出**：每日判断卡 + 次日复盘对话 + 月度"个人校准报告"。
- **数据需求**：低（仅需你的输入 + 价格数据做评分）。

### T3 · 信息情报团队（信息收集与分析）｜P2
- **目标**：从行业信息、世界新闻、政策、研报中筛出**值得关注**的，而不是堆一座信息垃圾山。
- **成员**：抓取、**信噪比过滤+去重**、摘要分类、定向推送。
- **核心纪律**：先定义"什么值得收"，再收。每条信息标来源、时间、影响判断。
- **输出**：每日情报简报（已分类、已去重）+ 行业知识沉淀，喂给 T1/T2。
- **数据需求**：中高（新闻/RSS/政策源）。

### T4 · 持仓管理团队（记录与管理）｜P0→P1
- **目标**：记录真实持仓与每笔操作，做绩效归因，但**绝不自动下单**。
- **成员**：持仓台账、交易流水、绩效归因、（只读）风险提示。
- **持仓台账字段**：标的 / 数量 / 成本 / 现价 / 浮盈亏 / 仓位占比 / 关联决策记录ID。
- **绩效归因**：把盈亏拆成"判断对不对"(连 T2) 和"执行对不对"(择时/仓位)。
- **数据需求**：低（手工录入 + 价格更新）。

## 3. 共享底座

| 模块 | 职责 | 不该做 | 阶段 |
|---|---|---|---|
| **数据层** | 抓取/清洗/落地：行情、财务、宏观、资金、新闻 | 不做实时交易接口 | P1 |
| **知识与记忆层** | wiki 索引 + **我的认知档案**（偏差、强项、风险偏好）| 不存无来源结论 | P1 |
| **总编/协调** | 派任务单、串流程、强制走反方+风险闸、出周报 | 不替我做决策 | P0 |
| **反方 + 风险闸** | 任何进入决策日志的想法必须过这两关 | 不可被跳过 | P0(角色)/P1(成熟) |
| **金融基础教练** | 横切：补概念、纠偏认知 | — | P2 |

## 4. 平台 / 存储 / 呈现层（回答你的飞书+WebUI+Git）

**真相源 = Git 仓库**。所有代码 + **结构化数据**（判断、持仓、决策、研究）都以文件形式（JSON/CSV/Markdown，后期可加 SQLite）存在 repo 里。好处：版本可追溯、多机 `git pull` 即用、未来 WebUI 直接读它。

| 层 | 角色 | 关系 |
|---|---|---|
| **Git 仓库** | 真相源（代码+结构化数据+文档） | 一切的根，多机同步靠它 |
| **飞书** | 云端**人读**资料库 | 定期从 repo **单向导出**人类要看的报告/知识；不做双向同步（避免冲突地狱） |
| **WebUI**（未来 P3） | 人机交互与操作前端 | 读写同一份 repo/DB，提供录判断、看复盘、管持仓的界面 |

> **现在为 WebUI 只需做一个决定**：所有状态用**结构化文件/SQLite**存，不要锁死在散文里。这样未来 WebUI/API 能直接挂上去，零返工。除此之外 WebUI 一概不提前建。

## 5. 数据源推荐（你可以去搞）

A 股个人投资者，按"先免费跑起来，再按需加稳定源"：

| 用途 | 首选 | 备选 | 说明 |
|---|---|---|---|
| 行情K线/基础财务/宏观 | **AkShare**（免费、开源、无需 token） | Baostock（稳定历史K线） | AkShare 覆盖最广，个人首选；缺点是源变动时偶尔失效 |
| 完整财务报表/稳定接口 | **Tushare Pro**（注册、积分制） | — | 需要可靠财报时上，积分可免费攒或付费 |
| 新闻/政策/研报 | **RSS + 财联社电报** | NewsAPI（世界新闻，含付费档） | T3 团队用；先 RSS 跑通再说 |

**建议**：P0/P1 阶段用 **AkShare 一个就够**（行情+财务+宏观全覆盖、零成本）。等 T1 深度分析对财报精度有要求了，再加 Tushare Pro。世界新闻等 T3 启动时再接。

## 8. 分阶段路线（按"低数据依赖、高日常价值"优先）

> 关键排序思想：**先做只需要"我自己输入"的部分**（判断、持仓），它们每天产生价值、还顺便积累自我提升所需的数据；数据饥渴的深度分析放后面。

| 阶段 | 名称 | 交付 | 为什么现在 |
|---|---|---|---|
| **P0** | 项目骨架 + 自我中心 | Git 项目 + 目录 + 模板 + **T2 判断校准** + **T4 持仓台账** + 总编/反方/风险（提示词） | 零外部数据依赖，每天能用，立刻养成习惯并攒数据 |
| **P1** | 数据 + 深度分析 | 数据层(AkShare) + **T1 个股深度分析** + 反方/风险闸成熟 + 知识/认知档案 | 有了数据，T1 才能跑；判断/持仓也能自动评分 |
| **P2** | 信息情报 + 飞书 | **T3 信息收集** + 信噪比过滤 + 飞书单向导出 + 金融教练 | 骨架稳后扩广度与云端沉淀 |
| **P3** | 平台化 + 量化 | **WebUI** + 校准分析看板 + 量化研究(防过拟合) | 风险最高、最重，最后做 |

**现在明确不做**：WebUI、量化回测、飞书双向同步、交易接口、组合自动化。

## 9. 下一步

P0 是地基且零数据依赖。建议下一步直接：**初始化 Git 项目 + 建目录 + 写模板 + 搭 T2 判断校准和 T4 持仓台账的最小可用版本**。完成后你今晚就能记录第一条判断、录入第一笔持仓，明天就能跑第一次复盘对话。数据源（AkShare）你可以并行去装，P1 才用得上。
